Promise 방식 vs Success/Fail URL (리다이렉트)

가장 혼란스러워하셨던 부분입니다. 결론부터 말씀드리면 모바일을 지원하려면 반드시 successUrl, failUrl을 사용하는 "리다이렉트 방식"을 써야 합니다.
왜 Promise 방식은 안 되나요?

    Promise 방식: 결제 버튼을 누르면 그 자리에서 응답을 기다립니다. (PC 웹 전용)

    리다이렉트 방식: 결제 버튼을 누르면 카드사 앱이나 다른 페이지로 이동했다가 다시 내 사이트로 돌아옵니다.

    문제점: 모바일에서는 카드사 앱(현대카드, 삼성카드 등)으로 화면이 전환됩니다. 이때 기존 브라우저의 JS 컨텍스트가 끊기기 때문에 Promise(.then)가 작동하지 않습니다.

    결론: successUrl과 failUrl을 설정하세요. 그래야 모바일 앱 결제 후 정상적으로 내 서비스로 돌아올 수 있습니다.
