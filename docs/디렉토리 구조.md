# Next js

## 핵심 파일 구조 및 역할
 * app/ (또는 src/app/): 라우팅의 핵심. 폴더가 곧 URL 경로가 됩니다.
   * layout.js: 공통 레이아웃 (네비게이션, 푸터 등).
   * page.js: 해당 경로의 UI (실제 화면).
   * loading.js: 로딩 상태 UI.
   * error.js: 에러 처리 UI.
   * route.js: API 엔드포인트 생성.
   * not-found.js: 404 페이지
 * public/: 정적 파일 (이미지, 폰트 등).
 * components/: 재사용 가능한 UI 컴포넌트 (버튼, 카드 등).
 * lib/ 또는 utils/: 공통 함수, API 클라이언트 등 유틸리티.
 * next.config.js: Next.js 프로젝트 설정.
 * package.json: 프로젝트 의존성 및 스크립트.
 * hooks/: 커스텀 React 훅 (예: useAuth, useFetch 등).

## 추천 기본 구조
```text
my-app/
├── src/
│   ├── app/           # 라우팅 (page.js, layout.js)
│   ├── components/    # 공통 컴포넌트 (Button, Header)
│   ├── hooks/         # 커스텀 훅
│   ├── lib/           # 라이브러리 설정, 유틸 함수
│   └── styles/        # 전역 스타일
├── public/            # 정적 파일
├── next.config.js
└── package.json
```

## 실무 구조
```text
src/
├─ app/                  # (Routing) URL과 매핑되는 페이지들
│  ├─ (auth)/            # 그룹화: URL에는 영향 없음 (로그인/회원가입)
│  │  ├─ login/page.tsx
│  ├─ payments/          # 결제 관련 페이지
│  │  ├─ success/page.tsx
│  │  └─ page.tsx        # /payments
│  ├─ layout.tsx         # 전역 레이아웃
│  └─ page.tsx           # 메인 페이지 (/)
│
├─ components/           # (UI 부품)
│  ├─ ui/                # 버튼, 인풋 등 아주 작은 단위 (Shadcn UI 스타일)
│  ├─ common/            # 헤더, 푸터 등 공통 레이아웃 부품
│  └─ payments/          # 결제 위젯 렌더링용 전용 컴포넌트
│
├─ hooks/                # 커스텀 훅 (ex: usePayment, useAuth)
├─ lib/                  # 외부 라이브러리 설정 (toss-sdk 초기화 등)
├─ services/             # API 호출 로직 (axios/fetch 등 백엔드 통신)
├─ types/                # TypeScript 타입 정의 파일
└─ utils/                # 공통 함수 (날짜 포맷팅, 숫자 콤마 등)
```
